# 性能测试报告

## 测试环境

- **CPU**: Intel/AMD (支持 AVX2 和 SSE4.2 指令集)
- **内存**: 充足可用内存 (>256MB)
- **存储**: SSD/NVMe 固态硬盘（对 I/O 并发测试至关重要）
- **操作系统**: Linux / macOS
- **编译参数**: `gcc -O3 -mavx2 -msse4.2 -fPIC -DSQLITE_ENABLE_FTS5 -DSQLITE_ENABLE_JSON1 -DSQLITE_ENABLE_RTREE`

## 基准测试结果预期

基于 SQLite 3.44.0 进行的各阶段对比：

### 1. 无锁写入队列 (Stage 1)
- 消除 WAL 日志写入锁争用，支持高并发无锁排队。
- 预期 **TPS 提升 2.5x - 3.5x** (高写入并发场景)。 

### 2. ARC 智能页缓存 (Stage 2)
- 改进原有的 LRU 替换策略，动态适应时序访问模式。
- 预期命中率提升导致读性能 **提升 1.5x - 2x**。

### 3. 异步刷盘机制 (Stage 3)
- 将同步 `fsync` 的调用降至最低。
- 对小事务性能 **提升 1.5x - 2x**，但需注意在系统崩溃时的极短时间（<100ms）的数据丢失风险。

### 4. SIMD 优化 (Stage 4)
- 引入 `AVX2` 加速页面校验和计算等密集型操作。
- 进一步降低 CPU 占用，整体事务性能附加 **1.2x - 1.5x** 的提升。
